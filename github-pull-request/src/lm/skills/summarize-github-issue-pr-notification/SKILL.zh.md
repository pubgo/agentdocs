# 总结问题

## 目的

给定一个 json GitHub 问题、PR 或通知，此技能会总结内容，提供要点和关键细节的简明概述。这项技能可以帮助用户快速理解问题的本质，而无需阅读整个内容。

## 用法

要使用此技能，请提供 GitHub 问题的 JSON 表示形式。该技能将提取相关信息并生成摘要，以捕获问题的要点和关键细节。

## 关于如何总结问题的提示

- 不输出代码。当您尝试总结 PR 更改时，请以文本格式进行总结。
- 将其他问题和 PR 的引用输出为 Markdown 链接。
- 如果评论引用了例如问题或 PR #123，则根据它是问题还是 PR 在摘要中输出以下任一内容：
    - [#123](https://github.com/${owner}/${repo}/issues/123)
    - [#123](https://github.com/${owner}/${repo}/pull/123)
- 评论应先与作者一起总结。例如：
    - @username：这是一条评论，总结了评论的要点。
- 如果内容包含 Markdown 格式的图像（例如！[alt text](image-url)），请始终将它们准确地保留在输出中。图像是重要的视觉内容，不应被删除或概括。
- 确保摘要至少与问题或 PR 以及评论和补丁（如果有）一样短或更短。
