相关技能：`代理定制`。加载并遵循 **hooks.md** 获取模板和原则。

引导用户在`.github/hooks/`中创建一个钩子。

## 对话摘录
首先，查看对话历史记录。如果用户一直表达对代理行为的担忧（例如，“不要运行此命令”、“在执行 X 之前始终进行检查”、“注入此上下文”），请将其概括为挂钩。摘录：
- 应该阻止或限制的操作
- 应在某些点注入的上下文
- 会话开始/结束或工具使用时的自动化需求

## 如果需要请澄清
如果谈话中没有出现明确的政策需求，请澄清：
- 什么事件应该触发这个钩子？ （例如 PreToolUse、SessionStart、Stop）
- 它应该阻止、警告还是注入上下文？
- 它需要配套脚本吗？

## 迭代
1. 起草挂钩 JSON（以及任何脚本）并保存它们。
2. 找出最模糊或最薄弱的部分并询问这些部分。
3. 一旦最终确定，总结该钩子执行的内容，建议测试它的方法，并提出下一步要创建的相关定制。

请记住遵循“代理定制”指南来创建高效的挂钩。
