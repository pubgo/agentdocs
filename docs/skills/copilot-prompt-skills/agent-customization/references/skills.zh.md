## 渐进式加载

1. **发现**（~100 个令牌）：代理读取“名称”和“描述”
2. **说明**（<5000 个标记）：在相关时加载 `SKILL.md` 正文
3. **资源**：仅在引用时加载附加文件

将文件引用保留在“SKILL.md”深一层。

## 斜线命令行为

在聊天中输入“/”后，技能和提示文件都会出现。

|配置|斜线命令 |自动加载 |
|---|---|---|
|默认（均省略）|是的 |是的 |
| `用户可调用： false` |没有 |是的 |
| `禁用模型调用：true` |是的 |没有 |
|两者都设置 |没有 |没有 |

## 何时使用

具有捆绑资产（脚本、模板、参考文档）的可重复的按需工作流程。

## 核心原则

1. **关键字丰富的描述**：包含发现的触发词
2. **渐进式加载**：将SKILL.md控制在500行以内；使用参考文件
3. **相对路径**：技能资源始终使用`./`
4. **自包含**：包含完成任务的所有程序知识

## 反模式

- **模糊的描述**：“有用的技能”无法发现
- **Monolithic SKILL.md**：所有内容都在一个文件中，而不是引用
- **名称不匹配**：文件夹名称与“名称”字段不匹配
- **缺少程序**：没有分步指导的描述
