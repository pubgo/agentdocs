## 核心原则

1. **关键字丰富的描述**：包含按需发现的触发词
2. **每个文件一个问题**：用于测试、样式、文档的单独文件
3. **简洁且可操作**：共享上下文窗口——保持专注
4. **展示，而不是讲述**：简短的代码示例胜过冗长的解释

## 反模式

- **模糊的描述**：“有用的编码提示”无法发现
- **过于宽泛的 applyTo**：`"**"` 内容仅与特定文件相关
- **复制文档**：复制自述文件而不是链接
- **混合关注点**：测试 + API 设计 + 样式在一个文件中
