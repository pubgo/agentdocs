# 渲染 GitHub 搜索结果

## 目的

获取 GitHub 搜索查询的结果（可能包括问题或拉取请求），并将其呈现为人类友好的 Markdown 表格格式。该技能从每个搜索结果中提取相关信息，并以易于阅读和理解的方式对其进行组织，使用户能够快速掌握每个问题或拉取请求的关键细节，而无需解析原始数据。

## 用法

要使用此技能，请传递来自 GitHub 搜索查询的原始搜索结果。然后，该技能将处理数据并生成一个降价表，该表总结每个问题或拉取请求的关键信息，例如标题、作者、标签、状态和任何其他相关详细信息。这使得用户更容易一目了然地查看和分析搜索结果。

## 如何呈现 GitHub 搜索结果

- 如果您有原始查询，请使用它来帮助确定要包含在表中的最重要字段。例如：
    - 如果查询包含特定标签，请确保不在表中包含该标签，因为所有结果都会包含该标签。
    - 如果查询包含“is:pr”，则重点关注与拉取请求相关的字段，例如“审核状态”和“合并状态”。
    - 包括与排序值相关的列（如果给定）。
    - 不要包含对于所有产生的问题都具有相同值的列。
- 始终包含项目编号和标题的列。将数字格式化为问题或 PR 的降价链接。例如：[#123](https://github.com/owner/repo/issues/123)
