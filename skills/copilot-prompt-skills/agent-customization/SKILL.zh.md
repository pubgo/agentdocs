## 创建过程

如果您需要探索或验证代码库中的模式，请使用只读子代理。如果提问工具可用，请使用它来采访用户并澄清需求。

创建任何自定义文件时请按照以下步骤操作。

### 1. 确定范围

询问用户他们想要在哪里进行定制：
- **工作区**：用于特定于项目、团队共享的自定义 → `.github/` 文件夹
- **用户配置文件**：用于个人、跨工作空间自定义 → `{{USER_PROMPTS_FOLDER}}/`

### 2. 选择正确的原语

使用上面的决策流程根据用户的需要选择适当的文件类型。

### 3. 创建文件

直接在适当的路径创建文件：
- 使用每个参考文件中的位置表
- 根据需要包括所需的前言
- 根据模板添加正文内容

### 4. 验证

创建后：
- 确认文件位于正确的位置
- 验证 frontmatter 语法（`---` 标记之间的 YAML）
- 检查“描述”是否存在且有意义

## 边缘情况

**说明与技能？** 这是否适用于“大多数”工作或“特定”任务？大多数 → 说明。具体→技能。

**技能与提示？** 两者都在聊天中显示为斜线命令（键入“/”）。具有捆绑资产的多步骤工作流程 → 技能。带输入的单一聚焦任务 → 提示。

**技能与自定义代理？** 所有步骤均具有相同的功能 → 技能。需要上下文隔离（子代理返回单个输出）或每个阶段不同的工具限制 → 自定义代理。**挂钩与指令？**指令*指导*代理行为（非确定性）。通过 shell 命令在诸如“PreToolUse”或“PostToolUse”之类的生命周期事件中挂钩“强制”行为——它们可以阻止操作、需要批准或确定性地运行格式化程序。请参阅[hooks 参考](./references/hooks.md)。

## 常见陷阱

**描述是发现表面。** “描述”字段是代理决定是否加载技能、指令或代理的方式。如果描述中没有触发短语，代理将找不到它。使用带有特定关键字的“Use when...”模式。

**YAML frontmatter 无提示失败。** 值中未转义的冒号、制表符而不是空格、“名称”与文件夹名称不匹配 — 所有这些都会导致无提示失败，且没有错误消息。始终引用包含冒号的描述：`描述：“在执行 X 时使用”`。

**`applyTo: "**"` 会烧毁上下文。** 这意味着“始终包含在每个文件请求中”——它会在每次交互时将指令加载到上下文窗口中，即使是不相关的。使用特定的 glob（`**/*.py`、`src/api/**`），除非该指令真正适用于所有文件。
