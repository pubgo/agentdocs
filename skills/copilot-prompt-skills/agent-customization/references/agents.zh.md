## 调用

- **手动**：聊天中的代理选择器
- **子代理**：基于“描述”匹配的父代理委托（当“推断”允许时）

## 核心原则

1. **单一角色**：每个代理有一个角色，负责集中职责
2. **最少的工具**：只包含角色需要的东西——过多的工具会稀释焦点
3. **清晰的界限**：定义代理不应该做什么
4. **关键字丰富的描述**：包括触发词，以便父代理知道何时进行委托

## 反模式

- **瑞士陆军特工**：工具太多，试图做所有事情
- **模糊的描述**：“乐于助人的代理人”并不能指导授权——要具体
- **角色混乱**：描述与身体角色不符
- **循环切换**：A→B→A，没有进度标准
