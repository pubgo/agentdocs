# [工作区说明](https://code.visualstudio.com/docs/copilot/customization/custom-instructions)

自动应用于整个工作区中所有聊天请求的准则。

## 文件类型（选择一种）

|文件 |地点 |目的|
|------|----------|---------|
| `副驾驶指令.md` | `.github/` |项目范围的标准（推荐，跨编辑器）|
| `代理.md` |根文件夹或子文件夹 |开放标准、monorepo 层次结构支持 |

**仅使用一个**，而不是同时使用。

## AGENTS.md 层次结构

对于 monorepos，目录树中最接近的文件优先：

@@代码块0@@

当不同区域需要不同的默认值时，请为 monorepos 使用嵌套的“AGENTS.md”文件。

## 模板

仅包括工作区受益的部分：

@@代码块1@@

对于大型存储库，请链接到详细文档而不是嵌入：“请参阅 docs/TESTING.md 了解测试约定。”

## 何时使用

- 适用于任何地方的通用编码标准
- 通过版本控制共享团队首选项
- 项目范围的要求（测试、文档）

## 核心原则

1. **默认情况下最小**：仅与*每个*任务相关的内容
2. **简洁且可操作**：每一行都应该指导行为
3. **链接，不要嵌入**：参考文档而不是复制
4. **保持最新**：实践发生变化时更新

## 反模式

- **使用两种文件类型**：同时具有`copilot-instructions.md`和`AGENTS.md`
- **厨房水槽**：一切而不是最重要的
- **复制文档**：复制自述文件而不是链接
- **明显的说明**：linter 已经强制执行的约定
